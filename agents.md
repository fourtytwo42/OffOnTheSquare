# Agents Guide

This file defines the default expectations for any agent working in this repository.

## Core Rules

1. Read the relevant project files before making changes.
2. Keep changes focused on the task at hand and avoid unrelated edits.
3. Do not overwrite or revert user changes unless explicitly asked.
4. After every major change, always create a git commit and push it to GitHub.
5. If a change cannot be safely verified, clearly note the risk and what still needs to be checked.

## Repo Notes

- `storefront/AGENTS.md` contains additional rules specific to the storefront app.
- When working inside a subdirectory with its own agent instructions, follow the more specific file in addition to this one.
